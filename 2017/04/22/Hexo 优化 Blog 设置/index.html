「 Template render error 」问题

1. Disqus 评论区

花了很长时间设置成功。期间因未能利用好「本地预览」功能而走入误区

`hexo clean && hexo g && hexo s`

之前对于 hexo s 与 hexo d 命令没有明确的区别意识，也不熟悉搭建 Blog 的体系。总是直接 deploy 然后多刷新几遍，结果让延迟效果把自己弄得混乱。幸好及时意识到错误，检查修改代码的工作量也不大。第一次加载 Disqus 时的延迟让我以为需要添加 Universal Code 才可实现，但如果使用 hexo s 指令，则会看到「评论区」已然在「非首页」的页面里出现。再次多处尝试添加 Universal Code 给我自己造成了很大的混乱，最后还得一一删除。

所以设置 Disqus 只需要在其官网创帐号，add to your site 然后设置相关 URL、shortname 等，再回到本地修改 _config.yml 文件。我分别在站点和 themes 里的两个 _config.yml 都添加修改了相关内容，前者添加shortname，后者在 disqus 字段里 enable 并填写 shortname 即可。

「分类」和「标签」可在 index.md 文件里写入 comments: false 来取消评论。

评论区域修改好了。我的 footer 位置又出了问题，「所有文章」里的 footer 都未能固定在底端，如果文章长度超出屏幕需要滚动阅览，footer 则会停留在原本的屏幕下方，也就是遮盖在文章中间。可以锁定是 post 相关文件的错误。搜索查询发现相关问题很少，相似经历更少且没找到有效solution（缩小范围到自己 code 问题），又尝试了删除改动 footer 未果，我意识到可能是因我之前乱添 Universal Code 遗留的问题，很大可能误删或者少删了内容。

于是我打开 NexT 主题在 GitHub 上的相关代码对照了两个文件，果然发现少删除一个 </div> 造成区域混乱。

问题解决。Disqus 成功添加。
